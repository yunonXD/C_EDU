
스택 Stack
    삽입과 삭제연산이 후입선출LIFO 자료구조 (한쪽에서만 데이터의 삽입 삭제가 이루어짐)
    새롭게 들어온 값이 Top 으로 변함
    pop은 top 을 가리키는 데이터를 스택에서 빼게 되어있음
    
    pop     : 데이터 삭제 및 확인 
    push    : 데이터 삽입
    top     : 데이터 단순 확인

    스탭은 깊이 우선 탑색 DFS , 백트래킹에 효과적 (LIFO --> 재귀함수와 똑같음)

    스택 , 재귀함수가 왜 둘 다 LIFO ? 그야.. 재귀도 들어간 순서대로 쌓이고 결과는 그의 반대로 나오니까..


큐 Queue
    선입선출(FIFO)
    흔히 말하는 줄서기
    추가는 (push) back 에서, 삭제는 front 에서 진행

    back : 가장 끝 데이터
    front : 앞의 데이터 

    ===================
    back            front
    ===================

    push : 삽입
    pop : 삭제

    너비 우선 탐색 BFS 에 주로 사용


우선순위 큐
    값이 들어간 순서에 상관없이 우선순위에 따라서 먼저 나오는 자료구조
    Max or min 을 유지해서 정렬해줌
